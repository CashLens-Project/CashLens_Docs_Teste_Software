# Documentação do CashLens - Teste de Software

Este repositório centraliza os documentos do projeto CashLens utilizados na disciplina de Teste de Software. Os arquivos foram organizados em formato PDF para facilitar a entrega, a revisão e a consulta pelos avaliadores.

## Objetivo

O objetivo desta documentação é reunir os principais artefatos produzidos durante o desenvolvimento e a validação do CashLens, incluindo documentos de visão, requisitos, planejamento de testes, histórias de usuário, casos de teste e relatório final.

## Estrutura dos Documentos

| Arquivo | Descrição |
| --- | --- |
| `documento_visão.pdf` | Apresenta a contextualização do sistema, público-alvo, problema tratado, objetivos, escopo e visão geral da solução. |
| `requisitos_de_software.pdf` | Consolida os requisitos funcionais e não funcionais definidos para o projeto CashLens. |
| `plano_de_teste.pdf` | Define a estratégia planejada para validação do sistema, incluindo tipos de teste, ambiente, ferramentas, riscos e critérios de aceitação. |
| `historia_de_usuario.pdf` | Descreve os principais cenários de uso do CashLens sob a perspectiva do usuário, como login, cadastro, recuperação de senha, fluxo de caixa, DRE, waterfall, operações e IA. |
| `casos_de_teste.pdf` | Apresenta casos de teste priorizados, com identificador, descrição, passos de execução, dados de entrada, saída esperada e critério de sucesso. |
| `relatorio_final.pdf` | Consolida a execução dos testes, evidências, resultados obtidos, lições aprendidas, conclusão e links dos repositórios do projeto. |
